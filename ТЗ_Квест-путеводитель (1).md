# Техническое задание
## на разработку мобильного приложения «Квест-путеводитель для туристов»

---

## 1. Общие сведения

**Наименование проекта:** Квест-путеводитель для туристов (Quest Guide for Tourists)

**Тип проекта:** Мобильное приложение с элементами геймификации и навигации.

**Назначение системы:** Приложение предназначено для организации самостоятельных туристических маршрутов в формате интерактивного квеста с использованием геолокации, заданий и системы достижений.

**Целевая аудитория:**
- Туристы (индивидуальные путешественники)
- Студенты и молодёжь
- Гости города
- Экскурсионные группы (self-guided формат)

---

## 2. Цель разработки

Создание мобильного приложения, которое:
- превращает обычную экскурсию в игровой квест
- повышает интерес к достопримечательностям
- обеспечивает самостоятельное изучение города
- использует GPS-навигацию + задания + систему наград

---

## 3. Основные задачи системы

Приложение должно обеспечивать:
- Отображение туристических маршрутов
- Навигацию пользователя по GPS
- Выполнение заданий на каждой точке маршрута
- Проверку ответов пользователя
- Начисление баллов и достижений
- Отображение истории прохождения
- Информацию о достопримечательностях
- Работу без экскурсовода (Self-Guided Tourism)

---

## 4. Функциональные требования

### 4.1 Регистрация и авторизация

Пользователь может:
- Зарегистрироваться (email / Google / Apple)
- Войти в систему
- Сохранить прогресс прохождения

### 4.2 Выбор квеста

Пользователь может:
- Выбрать город
- Выбрать маршрут
- Увидеть: длительность, сложность, расстояние, количество точек

### 4.3 Карта и навигация

Приложение должно:
- Отображать карту (Google Maps / OpenStreetMap)
- Показывать текущее местоположение
- Строить маршрут до следующей точки
- Уведомлять при приближении к объекту

### 4.4 Квест-механика

На каждой точке пользователь получает задание:
- Вопрос (текстовый)
- Загадку
- Поиск объекта
- Фото-задание
- Выбор ответа (тест)
- Ввод текста

После выполнения:
- Система проверяет ответ
- Открывает следующую локацию

### 4.5 Информационный блок

Каждая достопримечательность содержит:
- Описание
- Историческую справку
- Фото
- Аудиогид (опционально)

### 4.6 Система баллов и достижений

Приложение начисляет:
- Очки за правильные ответы
- Бонусы за скорость прохождения
- Достижения (Badges)
- Рейтинг пользователя

### 4.7 История прохождения

Пользователь может:
- Просматривать завершённые квесты
- Видеть статистику
- Повторять маршруты

Система состоит из:
- Мобильного клиента
- Серверной части (API)
- Базы данных маршрутов
- Админ-панели для добавления квестов

---

## 8. Административная панель

Администратор может:
- Создавать маршруты
- Добавлять задания
- Редактировать описание объектов
- Загружать фото/аудио
- Просматривать статистику пользователей

---

## 9. Требования к интерфейсу

Интерфейс должен быть:
- Современным
- Минималистичным
- Удобным для туриста «на ходу»
- С крупными кнопками
- Поддержка 2 языков: **RU / KZ**
